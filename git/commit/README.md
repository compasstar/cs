# Commit Convention

## ✅ 커밋 메시지

| 커밋 메시지                    | 설명                    |
|---------------------------|-----------------------|
| [CREATE] 로그인 페이지 UI 구현       | 새로운 기능 추가             |
| [UPDATE] 로그인 실패 시 UX 개선   | 기존 기능 변경              |
| [DELETE] 로그인 페이지 삭제    | 기존 기능 삭제               |
| [DOCS] README 설치가이드 수정    | 문서 변경                 |

> 4개만 사용하는 이유. UPDATE 의 구분이 모호하다. 현실적으로 UPDATE, REFACTOR, FIX 등 동시에 이루어진다. 구분하기 어렵고, 뒤에 커밋 메시지로 간단하게 명시해주면 충분하다

## ✅ 규칙
1. 현재형 동사로 문장을 구성한다 (추가함(X) -> 추가 또는 구현)
2. 마침표는 찍지 않는다
3. 본문이 필요하다면 제목 아래 한 줄 띄우고 작성한다
